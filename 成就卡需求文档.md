# 🎲 TRPG 成就卡片生成器 (TRPG Achievement Forge) - 产品需求文档

## 1. 项目概述
**项目名称**：TRPG 成就卡片锻造炉 (TRPG Achievement Forge)
**项目目标**：开发一款免费、开源、易用的纯前端 Web 工具，帮助 TRPG 玩家和主持人快速生成精美、具有纪念意义的“横幅式成就卡片”。
**核心特性**：实时预览、闪烁动态金字、可选自定义元素、纯本地数据闭环（无需后端）、支持导出 PNG 与 JSON、支持 JSON 逆向导入编辑。
**部署平台**：GitHub Pages（纯静态网站）。

## 2. 目标用户
*   **游戏主持人 (DM/KP/ST)**：为模组结团、玩家高光时刻或“大失败/大成功”整活颁发纪念卡片。
*   **跑团玩家 (PL)**：记录自己角色的传奇经历或搞笑瞬间。

## 3. 功能需求 (Functional Requirements)

### 3.1 信息输入模块 (编辑器)
提供一个表单供用户填写成就信息：
*   **成就图标 (Icon)**：[非必填] 支持本地上传图片，需自动裁剪/缩放为正方形。
*   **成就名称 (Title)**：[必填] 单行文本，限制字数（如20字内）。
*   **获得者 (Recipient)**：[必填] 玩家或角色名称。
*   **成就描述 (Description)**：[必填] 多行文本，支持自动换行，说明获得成就的原因。
*   **游戏名称 (Game Name)**：[非必填] 如“克苏鲁的呼唤：星之吸血鬼”、“D&D：博德之门”。
*   **颁发者 (Issuer)**：[非必填] 如“KP阿伟”或“命运女神”。
*   **时间戳 (Date)**：[非必填] 默认获取当前日期，允许手动修改。
*   **背景选择 (Background)**：提供几种预设的高质量、低干扰背景（如：暗黑羊皮纸、赛博朋克深色网格、星空、极简深灰），并支持自定义颜色。

### 3.2 实时预览模块 (画布)
*   **所见即所得**：左侧/上方修改表单，右侧/下方实时更新卡片样式。
*   **条件渲染**：当非必填项（Icon、游戏名称、颁发者、时间戳）为空时，卡片上对应的区域自动折叠隐藏，不留白、不占位，保持排版紧凑。

### 3.3 存储与导入导出模块
*   **导出为 PNG**：一键将卡片 DOM 节点转化为高清 PNG 图片并下载。
*   **导出为 JSON**：将当前填写的表单数据序列化为 `.json` 文件下载，作为工程文件。
*   **导入 JSON**：读取本地 `.json` 文件，解析后反填到输入表单中，恢复上次编辑状态。
*   **本地存储 (Local History)**：每次点击“保存/导出”时，自动将数据存入浏览器的 `localStorage` 或 `IndexedDB`，提供一个“历史记录”侧边栏，点击可随时恢复过往做过的成就。

## 4. 视觉与 UI/UX 需求 (UI/UX Requirements)

### 4.1 卡片排版设计 (横轴长条形)
*   **比例**：建议采用 `4:1` 或 `5:1` 的横轴比例（如：`1000px * 200px` 或 `1200px * 250px`）。
*   **布局**：
    *   **最左侧**：成就 Icon（如有），圆形或带金属边框的圆角矩形。
    *   **左中侧 (核心区)**：成就名称（大字号）、获得者名称（中字号，如：“授予：亚瑟·潘德拉贡”）。
    *   **中右侧**：成就描述（中/小字号，清晰易读）。
    *   **右下角/边角 (备注区)**：时间、游戏名、颁发者。字体较小，透明度较高（如 60%），不喧宾夺主。

### 4.2 字体与特效
*   **闪烁的金色字体 (核心视觉)**：
    成就名称必须使用 CSS 实现动态鎏金闪烁效果。
    *(技术提示：使用 `background: linear-gradient` 搭配 `background-clip: text; color: transparent;`，并通过 `@keyframes` 让背景位置 `background-position` 不断平移来实现波光粼粼的动态效果。)*
*   **字体排印**：建议引入 1-2 款开源美观的中文字体（如 霞鹜文楷 LXGW WenKai，增添 TRPG 叙事感）。
*   **无视觉干扰背景**：背景以暗色调为主（Dark Mode），辅以微弱的纹理（Noise/Texture），确保文字的高对比度和清晰度。

## 5. 技术架构与选型 (Technical Stack)

为了能够完美免费地部署在 GitHub Pages，且不依赖任何后端服务器：

*   **前端框架**：`Vue 3` 或 `React` (搭配 Vite 构建工具，开发极速，打包轻量)。
*   **样式框架**：`Tailwind CSS` (方便快速搭建响应式界面和暗黑模式)。
*   **图片生成核心库**：`html-to-image` 或 `html2canvas` (推荐前者，在现代浏览器中对现代 CSS 特性支持更好，用于将预览区的 HTML 转为 PNG)。
*   **数据存储**：
    *   `Pinia` (Vue) / `Zustand` (React) 进行状态管理。
    *   `localForage`：用于封装 IndexedDB，替代 localStorage。因为用户可能会上传 Icon，图片转 Base64 后体积较大，localStorage 只有 5MB 容量容易爆满，IndexedDB 容量极大且支持异步。
*   **图标库**：`Lucide Icons` 或 `Phosphor Icons`（用于界面UI的按钮图标）。

## 6. 数据结构设计 (JSON Schema)

导出的 JSON 文件以及存入 IndexedDB 的数据结构设计如下：

```json
{
  "id": "1698765432100", 
  "version": "1.0",
  "title": "直面古神而不疯",
  "recipient": "调查员：约翰·史密斯",
  "description": "在目击了克苏鲁的虚影后，连续三次通过了极难的理智（SAN）检定，展现出了钢铁般的意志。",
  "iconBase64": "data:image/png;base64,iVBORw0KGgo...", 
  "metadata": {
    "gameName": "克苏鲁的呼唤：阴影之海",
    "issuer": "KP 无情的骰子",
    "date": "2023-10-31"
  },
  "style": {
    "backgroundTheme": "dark-cthulhu"
  }
}
```
*注：上传的图片需在前端通过 `FileReader` 转换为 Base64 字符串，以便打包在 JSON 中实现完整的本地化。*

## 7. 部署与开源计划 (Deployment & Open Source)

1.  **代码托管**：在 GitHub 创建公开仓库 `trpg-achievement-forge`。
2.  **持续集成 (CI/CD)**：配置 `GitHub Actions`。当代码推送到 `main` 分支时，自动执行 `npm run build`，并将 `dist` 文件夹发布到 `gh-pages` 分支。
3.  **GitHub Pages 开启**：在仓库 Settings -> Pages 中选择 `gh-pages` 分支作为页面来源。
4.  **README 设计**：
